# Enterprise Standards

This memory pack contains canonical enterprise standards for AI assistants and development teams.

## Purpose

Provides authoritative guidance on approved patterns, tools, and practices across the organization.

## Structure

Standards are organized by domain:

- `enterprise/ado/` — Azure DevOps pipeline standards
- `enterprise/api/` — REST API design standards
- `enterprise/bicep/` — Bicep infrastructure as code standards
- `enterprise/docker/` — Docker container standards
- `enterprise/dotnet/` — .NET application standards
- `enterprise/observability/` — Logging, monitoring, and tracing standards
- `enterprise/security/` — Security and compliance standards
- `enterprise/terraform/` — Terraform module versioning standards
- `enterprise/testing/` — Testing and quality gate standards

Versioned snapshots are available under `v3/` and `v4/` for standards that have been through major revisions.

## Key Files

- `manifest.yaml` — pack metadata, ownership, classification, and embedding config
- `instructions.md` — how to use and reference these standards
