# Enterprise Standards Instructions

## How to Use These Standards

When providing guidance on any topic covered by an enterprise standard — including infrastructure, pipelines, security, API design, application development, containerization, observability, and testing — always check the relevant standard first. Enterprise standards take precedence over repo-level conventions.

## Classification

All standards in this pack are classified as `internal` unless otherwise noted. Do not share standard content with external personas.

## Versioning

Standards are versioned. Always reference the specific version when citing a standard. If no version is specified, use the latest available version.
